## 3. 安全性需求

### 3.1 认证授权

| 项目 | 要求 |
|------|------|
| 密码强度 | 最少8位，含大小写字母和数字 |
| 密码加密 | bcrypt + 随机盐 |
| JWT过期时间 | Access Token 15分钟，Refresh Token 7天 |
| 登录失败锁定 | 连续5次失败锁定30分钟 |
| 会话超时 | 30分钟无操作自动登出 |

### 3.2 数据安全

| 项目 | 要求 |
|------|------|
| 敏感数据加密 | 身份证号、银行卡号等需加密存储 |
| 传输加密 | 全站HTTPS (TLS 1.2+) |
| 脱敏规则 | 手机号中间4位***，身份证号***显示 |
| 日志脱敏 | 密码、Token等日志中不显示 |

### 3.3 安全审计

| 项目 | 要求 |
|------|------|
| 操作日志 | 保留180天 |
| 登录日志 | 保留360天 |
| 交易日志 | 保留5年 |
| 审计日志 | 记录所有敏感操作 |

### 3.4 防护措施

| 措施 | 要求 |
|------|------|
| SQL注入防护 | 参数化查询 |
| XSS防护 | 输入输出转义 |
| CSRF防护 | Token验证 |
| 接口限流 | 单IP 100次/分钟 |
| DDoS防护 | 流量清洗 |
